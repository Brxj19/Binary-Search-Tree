Performance Notes

`BinarySearchTree` is a plain Binary Search Tree, so the cost of most operations depends on tree height. When the tree is shallow, lookups and insertions are fast. When the tree becomes tall and skewed, those same operations degrade toward linear time.

Complexity Table

Operation                 Average      Worst
Search                    O(log N)     O(N)
Insert                    O(log N)     O(N)
Erase                     O(log N)     O(N)
lower_bound / upper_bound O(log N)     O(N)
Traversal                 O(N)         O(N)
size / empty              O(1)         O(1)

Why Tree Shape Matters

If values are inserted in sorted order, the tree can become chain-like:

1
 \
  2
   \
    3
     \
      4

Practical Guidance

  • Use this library when you want a straightforward educational or lightweight BST.
  • Use height() to observe whether insertion order is creating a tall tree.
  • If guaranteed logarithmic complexity is required, prefer a self-balancing tree or a standard container such as std::set.
